Orchestrate AI agents

Maestro turns a YAML file into a parallel, dependency-aware pipeline of AI agents and shell commands — with DAG scheduling, context passing, budgets, quality gates and deterministic logs. One CLI, seven engines.

⬇ Install from PyPI View on GitHub
pip install maestro-ai-cli
Engines: Codex · Claude · Gemini · Copilot · Qwen · Ollama · Llama
pipeline.yaml
version: 1
name: ship-feature
tasks:
  - id: implement
    engine: claude
    model: sonnet
    prompt: "Implement the /export endpoint"

  - id: review
    engine: codex
    depends_on: [implement]   # runs after
    context_from: [implement]  # sees its output
    prompt: "Review the diff for bugs"

  - id: tests
    command: "pytest -q"
    depends_on: [implement]   # parallel w/ review

Why Maestro

A control plane for multi-agent work

Declare the pipeline once. Maestro handles scheduling, context, cost, quality and reproducibility — so you stop gluing engine CLIs together with bash.

🧬

DAG scheduling

Dependencies resolve automatically; independent tasks run in parallel via a thread pool. Cycles are caught at validation.

🔀

7 engines, 1 schema

Codex, Claude, Gemini, Copilot, Qwen, Ollama and Llama — plus raw shell. Mix them in one plan; route by cost or complexity.

🧠

11 context modes

Pass upstream output downstream — raw, summarized, map-reduce, recursive, layered, BM25-selective, structural, council, knowledge-graph, codebase-map or SCIP.

✅

Quality gates

Per-task verify_command, typed asserts, and LLM judges (rubrics, G-Eval, debate, quorum) that can fail, warn or retry.

💰

Cost & token budgets

Per-run pricing tables, soft max_cost_usd limits, cross-run ledgers and warnings before you overspend.

🛡️

Security built in

23 audit rules (maestro audit), untrusted-context taint tracking, prompt-injection containment and tool allow-lists.

🔁

Resilient by default

Retries with backoff, model escalation, cross-engine fallback, circuit breakers and adaptive re-planning on failure.

📦

Deterministic artifacts

Every run writes JSON manifests, transcripts and hash-chained event logs you can maestro verify, diff and replay.

🔌

Protocols & SDK

MCP server, AG-UI & OpenTelemetry export, a Web UI, an interactive shell, and a typed Python SDK (py.typed).

How it works

From file to pipeline in three steps

01

Declare

Write a version: 1 plan: tasks, engines, prompts, dependencies and gates. Validate it offline with maestro check.

02

Run

maestro run plan.yaml --parallel. Maestro schedules the DAG, shells out to each engine, passes context and tracks cost live.

03

Inspect

Get a run_summary.md, structured manifests and replayable logs. Compare runs with diff, trace failures with blame.
